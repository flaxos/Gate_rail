GateRail Stage 2 — Godot Client Wireframe

Low-fi wireframe for the MVP operations client. Backend stays authoritative — Godot reads JSON snapshots, sends commands. Three layout directions, panel callouts, and a build order for the next few sprints.

v0.1 · WIP
what we're exploring

Three takes on the same MVP data. All three read the same snapshot (tick · worlds · nodes · links · trains · schedules · orders · contracts · finance · reputation) and send the same four commands today. The layouts just differ in where attention sits: A spreads everything at once (sim-classic), B pushes the map forward with a bottom console, C reveals one concern at a time through a rail of drawers.

Slice 1 — HUD + map + schedule toggles first

  • Top HUD: tick / cash / net-this-tick / reputation / bridge pill / play-pause-step controls
  • Map: worlds, links (rail vs gate distinction), nodes around each world, train markers at node_id
  • Left panel: schedule rows with ON/OFF pill + toggle button → SetScheduleEnabled
  • Alert strip: last command result + any bridge error

Exit: you can watch the sim tick, toggle a schedule, see the effect on the map. This is a thin skin over Sprint 10 and finishes Sprint 11's HUD-and-schedules half.

Slice 2 — Dispatch + pending orders + contracts second

  • Dispatch form with live-populated train/origin/dest/cargo selects from the snapshot
  • Pending orders panel with per-row cancel → CancelOrder
  • Right panel: contracts list with progress bars, due tick, reward/penalty, status pill
  • Alert strip gains disruption chips (pulled from links[].disrupted / disruption_reasons)

Exit: the Sprint 8/9 scenario is fully playable from the client — no terminal. Finishes Sprint 11.

Slice 3 — Selection + inspection polish third

  • Click world / node / link / train → inspector (starts as a floating card; later becomes drawer)
  • Contract click → focuses map on the relevant route
  • In-transit train visibility: interpolate position along link by travel_ticks progress
  • Congestion / power overlays as toggleable map layers

Exit: the client is readable at a glance during a 20-minute playtest. No new backend commands required.

Slice 4 — Construction slice 1 hooks later

  • Build-mode toggle on the map cursor; station + rail tools
  • New commands wired through the bridge (names TBD, validated server-side)
  • Build details drawer: cost / power / errors

Aligns with Phase 2 Sprint 12. Do not start until slices 1-3 are playtested.

Slice 5 — Construction slice 2 · gates, trains, schedules future

  • Gate hub + gate link placement with power feedback
  • Train purchase + schedule creation UI
  • Fleet subpanel

Responsive / window notes

Desktop target: 1440×900 comfortable, 1280×720 minimum. No real mobile — but the Layout B bottom console collapses well to a tall narrow window (≥ 960 px wide) for side-by-side work with the terminal. Under that, panels stack in order: HUD → map → alerts → contracts → schedules → dispatch → orders. Always keep HUD and alerts pinned.
